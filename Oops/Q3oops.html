OBJECT ORIENTED PROGRAMMING THROUGH JAVA LAB

Q3. Write a Java program that creates a user interface to perform integer divisions. The user enters two numbers in the text fields, Num1 and Num2. The division of Num1 and Num 2 is displayed in the Result field when the Divide button is clicked. If Num1 or Num2 were not an integer, the program would throw a Number Format Exception. If Num2 were Zero, the program would throw an Arithmetic Exception. Display the exception in a message dialog box.

Aim: To develop a Java program that creates a user interface to perform integer divisions. The program should allow the user to input two integers, Num1 and Num2, in text fields and display the result of their division in another text field when the Divide button is clicked. The program should also handle potential exceptions, such as Number Format Exception and Arithmetic Exception, and display them in a message dialog box.

System and Software Requirements: A Computer with any Operating system with javac compiler installed in it.

Description:
The Java program can be developed using Swing API for creating a user interface with text fields, a button and a label to display the result. The program should include an action listener to handle the button click event and perform the division operation on the input values entered by the user.
The following steps can be taken to implement the program:
Step 1: Create a Java class with a main method that will be responsible for launching the program.
Step 2: Define the graphical user interface using the Swing API. This can be done by creating a JFrame with two text fields labeled Num1 and Num2, a button labeled Divide and a label labeled Result.
Step 3: Attach an action listener to the Divide button. The action listener should read the values from the Num1 and Num2 text fields, perform the division operation and display the result in the Result label.
Step 4: Add exception handling to the program. If Num1 or Num2 are not integers, the program should throw a NumberFormatException. If Num2 is zero, the program should throw an ArithmeticException. These exceptions can be caught and handled by displaying the exception message in a message dialog box.
Step 5: Test the program by entering valid and invalid input values in the Num1 and Num2 text fields. Verify that the program is able to handle the exceptions as expected and display the exception message in the message dialog box.
Overall, this experiment theory outlines the steps to create a Java program with a user interface that allows users to perform integer divisions. By implementing exception handling, the program is able to handle invalid input values and provide feedback to the user in the form of a message dialog box.



Examples:
Suppose we have a Java program with a user interface that asks the user to enter two integer values, x and y, and then performs the division operation x / y. The program should display the result of the division in a label on the user interface.
To implement this program, we can follow the steps outlined in the experiment theory:
Step 1: Create a Java class with a main method that launches the program.
Step 2: Create a JFrame with two text fields labeled "x" and "y", a button labeled "Divide", and a label labeled "Result".
Step 3: Attach an action listener to the "Divide" button that reads the values from the "x" and "y" text fields, performs the division operation x / y, and displays the result in the "Result" label.
Step 4: Add exception handling to the program. If the user enters a non-integer value in either the "x" or "y" text field, the program should throw a NumberFormatException. If the user enters a zero value in the "y" text field, the program should throw an ArithmeticException. These exceptions can be caught and handled by displaying the exception message in a message dialog box.
Step 5: Test the program by entering valid and invalid input values in the "x" and "y" text fields. Verify that the program is able to handle the exceptions as expected and display the exception message in the message dialog box.
Overall, this theoretical example demonstrates how we can create a Java program with a user interface that performs integer division and handles exceptions to provide feedback to the user.

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;
public class IntegerDivisionUI implements ActionListener {
private JFrame frame;
private JLabel num1Label, num2Label, resultLabel;
private JTextField num1Field, num2Field, resultField;
private JButton divideButton;
public IntegerDivisionUI() {
// create UI components
num1Label = new JLabel("Num1:");
num2Label = new JLabel("Num2:");
resultLabel = new JLabel("Result:");
num1Field = new JTextField(10);
num2Field = new JTextField(10);
resultField = new JTextField(10);
resultField.setEditable(false);
divideButton = new JButton("Divide");
divideButton.addActionListener(this);
// create UI layout
JPanel panel = new JPanel(new GridLayout(4, 2));
panel.add(num1Label);
panel.add(num1Field);
panel.add(num2Label);
panel.add(num2Field);
panel.add(resultLabel);
panel.add(resultField);
panel.add(divideButton);
// create frame and add panel
frame = new JFrame("Integer Division");
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
frame.add(panel);
frame.pack();
frame.setVisible(true);
}
public void actionPerformed(ActionEvent e) {
try {
// get input values
int num1 = Integer.parseInt(num1Field.getText());
int num2 = Integer.parseInt(num2Field.getText());
// perform division
int result = num1 / num2;
// display result
resultField.setText(Integer.toString(result));
} catch (NumberFormatException ex) {
// handle invalid input
JOptionPane.showMessageDialog(frame, "Invalid input: " + ex.getMessage(), "Error", JOptionPane.ERROR_MESSAGE);
} catch (ArithmeticException ex) {
// handle division by zero
JOptionPane.showMessageDialog(frame, "Division by zero: " + ex.getMessage(), "Error", JOptionPane.ERROR_MESSAGE);
}
}
public static void main(String[] args) {
new IntegerDivisionUI();
}
}
