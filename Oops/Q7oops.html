Q7. Write a Java program to create an abstract class named Shape that contains two integers and an empty method named print Area ( ). Provide three classes named Rectangle, Triangle, and Circle such that each one of the classes extends the class Shape. Each one of the classes contains only the method print Area ( ) that prints the area of the given shape.

Aim: To create a Java program that demonstrates the use of an abstract class named Shape and its derived classes, Rectangle, Triangle, and Circle. The program will create an abstract class named Shape that contains two integer variables and an empty method named printArea(). The derived classes will extend the Shape class and implement their version of the printArea() method to print the area of the respective shape.

System and Software Requirements: A Computer with any Operating system with javac compiler installed in it.

Description:
Java is an object-oriented programming language that supports the creation of abstract classes, which are classes that cannot be instantiated but serve as blueprints for other classes. The Shape class in this scenario is an abstract class that contains two integers and an empty method named printArea().
The two integers in the Shape class likely represent the dimensions of the shape, such as its length and width or radius, depending on the specific shape. The printArea() method, being empty, serves as a placeholder for the area calculation algorithm that will be implemented by the subclasses.
The Rectangle, Triangle, and Circle classes are subclasses of the Shape class, which means they inherit the properties and methods of the Shape class. Each one of these classes contains only the printArea() method that calculates and prints the area of the given shape.
For example, the Rectangle class would contain a printArea() method that multiplies the length and width of the rectangle to calculate its area. Similarly, the Triangle class would contain a printArea() method that uses the formula for calculating the area of a triangle based on its base and height. Finally, the Circle class would contain a printArea() method that uses the formula for calculating the area of a circle based on its radius.
By extending the Shape class, these three classes can implement their specific area calculation algorithms while inheriting the properties and methods of the Shape class. This results in a more organized and maintainable code structure that reduces code duplication and promotes code reuse.



Examples:
Let's say we are creating a program that models different types of vehicles. We can define an abstract class named Vehicle that contains some common properties and methods that are shared by all types of vehicles, such as:
A String variable named make that represents the make of the vehicle (e.g. Ford, Toyota, etc.)
An int variable named year that represents the year the vehicle was manufactured
A boolean variable named isRunning that indicates whether the vehicle's engine is running or not
A void method named startEngine() that starts the vehicle's engine
A void method named stopEngine() that stops the vehicle's engine
Since not all vehicles have the same number of wheels, we can define an abstract method named getNumberOfWheels() that each subclass must implement. For example, a car class would return 4 while a motorcycle class would return 2.
We can then define different subclasses of the Vehicle class that represent specific types of vehicles, such as Car, Motorcycle, Truck, and Boat. Each subclass would implement the getNumberOfWheels() method and provide its own specific implementation of the startEngine() and stopEngine() methods.
For example, the Car class might have a startEngine() method that turns on the ignition and a stopEngine() method that turns off the ignition. The Motorcycle class might have a startEngine() method that kicks the starter and a stopEngine() method that cuts off the fuel supply.
By defining an abstract class and abstract method, we ensure that each subclass provides its own implementation while still sharing some common properties and methods defined in the abstract class. This results in a more organized and maintainable code structure that reduces code duplication and promotes code reuse.

import java.util.*;
abstract class shape
{
int x,y;
abstract void area(double x,double y);
}
class Rectangle extends shape
{
void area(double x,double y)
{
System.out.println("Area of rectangle is :"+(x*y));
}
}
class Circle extends shape
{
void area(double x,double y)
{
System.out.println("Area of circle is :"+(3.14*x*x));
}
}
class Triangle extends shape
{
void area(double x,double y)
{
System.out.println("Area of triangle is :"+(0.5*x*y));
}
}
public class AbstactDDemo
{
public static void main(String[] args)
{
Rectangle r=new Rectangle();
r.area(2,5);
Circle c=new Circle();
c.area(5,5);
Triangle t=new Triangle();
t.area(2,5);
}
}

interface Shape {
void printArea();
}
class Rectangle implements Shape {
private int length;
private int width;
public Rectangle(int length, int width) {
this.length = length;
this.width = width;
}
public void printArea() {
int area = length * width;
System.out.println("Area of rectangle: " + area);
}
}
class Triangle implements Shape {
private int length;
private int width;
public Triangle(int length, int width) {
this.length = length;
this.width = width;
}
public void printArea() {
double area = 0.5 * length * width;
System.out.println("Area of triangle: " + area);
}
}
class Circle implements Shape {
private int radius;
public Circle(int radius) {
this.radius = radius;
}
public void printArea() {
double area = Math.PI * radius * radius;
System.out.println("Area of circle: " + area);
}
}
public class Main {
public static void main(String[] args) {
Shape rectangle = new Rectangle(10, 20);
Shape triangle = new Triangle(10, 20);
Shape circle = new Circle(10);
rectangle.printArea();
triangle.printArea();
circle.printArea();
}
}
