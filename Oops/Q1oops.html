OBJECT ORIENTED PROGRAMMING THROUGH JAVA LAB

Q1. Use Eclipse or Net beans platform and acquaint with the various menus. Create a test project, add a test class, and run it. See how you can use auto suggestions, auto fill. Try code formatter and code refactoring like renaming variables, methods, and classes. Try debug step by step with a small program of about 10 to 15 lines which contains at least one if else condition and a for loop.

Aim: To familiarize oneself with the various menus and features of the Eclipse or NetBeans platform, including auto suggestions, auto fill, code formatter, and code refactoring. Additionally, to practice using the debug feature by stepping through a small program of about 10 to 15 lines which contains at least one if-else condition and a for loop.

System and Software Requirements: A Computer with any Operating system with javac compiler installed in it.

Description:
The Eclipse and NetBeans platforms are Integrated Development Environments (IDEs) used for developing and running Java programs. These platforms provide a range of menus and features that can help developers create, test, and debug their code efficiently. In this experiment, we aim to explore the various menus and features of the Eclipse or NetBeans platform and use them to create, run, and debug a test project.
Creating a Test Project:
To create a test project, we will first launch Eclipse or NetBeans and select "New Project" from the "File" menu. We will then choose "Java Project" and enter a name for the project. Once the project is created, we will add a new class to the project by selecting "New Class" from the "File" menu.
Using Auto Suggestions and Auto Fill:
Auto suggestions and auto fill are features that can help speed up coding by providing suggestions for commonly used code snippets. Eclipse and NetBeans offer this feature, and we can test it by typing the first few letters of a command or method name and then pressing "Ctrl + Space" to see the available options. This feature can also help with the syntax of the code.
Using Code Formatter:
The code formatter feature can help keep code consistent and readable. To test this feature, we will select "Source" from the top menu bar and then choose "Format" from the dropdown menu. This will format the code to follow the specified formatting rules.
Using Code Refactoring:
Code refactoring is the process of modifying code without changing its behavior. In Eclipse and NetBeans, we can use the "Refactor" option in the context menu to perform tasks such as renaming variables, methods, and classes. We can test this feature by selecting a variable or method name, right-clicking, and selecting "Refactor." We can then choose to rename the selected item.
Debugging the Program:
The debug feature is an essential tool for finding and fixing errors in code. To test this feature, we will create a small program of about 10 to 15 lines, containing an if-else condition and a for loop. We will then set a breakpoint in the code and run the program in debug mode. We can step through the program line by line, checking the values of variables at each step. When the program reaches the if-else condition, we can use the debug feature to ensure that the condition is executed correctly. We can then continue stepping through the program until the end of the loop is reached.
Conclusion:
By testing the various menus and features of the Eclipse or NetBeans platform, we can become more familiar with the IDE and its capabilities. Auto suggestions, auto fill, code formatter, code refactoring, and debugging are essential features that can help make coding more efficient and less prone to errors. Understanding how to use these features can help developers create high-quality code in less time.


Examples:
Suppose we want to create a simple Java program that calculates the sum of all even numbers between 1 and 10. We can use Eclipse or NetBeans to create a new project and a new class file called "SumEvenNumbers".
We can then use the auto-fill and auto-suggest features to quickly create the skeleton code for our program. For example, we can start by typing "public static void main" and then press "Ctrl + Space" to auto-fill the rest of the code.
Next, we can use the code formatter to ensure that our code is well-organized and easy to read. We can select the code and then choose "Format" from the "Source" menu to apply the formatting rules.
To calculate the sum of even numbers, we can use a for loop that iterates over all the numbers between 1 and 10. Inside the loop, we can use an if-else condition to check if the current number is even or odd. If it is even, we add it to the sum variable.
Finally, we can use the debugging feature to step through our code and ensure that it is working correctly. We can set a breakpoint inside the for loop and then run the program in debug mode. We can then step through the code line by line, checking the values of variables at each step.
Once we are satisfied that the program is working correctly, we can run it normally and see the output. In this example, the program should output the sum of even numbers between 1 and 10, which is 30.
Overall, this example demonstrates how we can use the various menus and features of Eclipse or NetBeans to create, test, and debug a simple Java program.

public class DebugExample {
public static void main(String[] args) {
int sum = 0;
for (int i = 1; i <= 10; i++) {
if (i % 2 == 0) {
sum += i;
} else {
sum -= i;
}
}
System.out.println("The sum is: " + sum);
}
}
