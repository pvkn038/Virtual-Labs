Q2. Design multi-dimensional data models namely Star, snowflake and Fact constellation schemas for anyone enterprise (ex. Banking, Insurance, Finance, Healthcare, Manufacturing, Automobile, etc.).

System and Software Requirements: A Computer with any WEKA Tool with installed in it.

Description:
In this scenario, we will focus on some key dimensions and facts relevant to a banking system.

1. Star Schema:

  • Customer Dimension: Contains customer-related information.
  • Account Dimension: Contains account-related information.
  • Date Dimension: Contains date-related information.
Facts table :

  • Transaction Fact: Contains transaction-related information such as transaction amount, transaction type, and reference to the customer, account, and date dimensions.
The Star Schema represents a simple and denormalized structure with a central fact table surrounded by dimensional tables. It provides easy querying and good performance for aggregations.

2.Snowflake Schema

  • Customer Dimension: Contains customer-related information.
  • Account Dimension: Contains account-related information.
  • Date Dimension: Contains date-related information.
  • Branch Dimension: Contains branch-related information.
Facts table :

  • Transaction Fact: Contains transaction-related information such as transaction amount, transaction type, and reference to the customer, account, and date dimensions.
The Snowflake Schema expands on the Star Schema by normalizing the dimensional tables. In this example, the branch dimension is further normalized into a separate table. This schema can provide better data integrity but may result in more complex queries.

3.Fact Constellation Schema:

  • Customer Dimension: Contains customer-related information.
  • Account Dimension: Contains account-related information.
  • Date Dimension: Contains date-related information.
  • Branch Dimension: Contains branch-related information.
Facts table :

  • Transaction Fact: Contains transaction-related information such as transaction amount, transaction type, and reference to the customer, account, and date dimensions.
  • Facts table :

    • Transaction Fact: Contains transaction-related information such as transaction amount, transaction type, and reference to the customer, account, and date dimensions.
The Fact Constellation Schema, also known as a Galaxy Schema, consists of multiple fact tables sharing common dimension tables. It allows for more flexibility and accommodates different types of facts within the same data model.

These are just examples of data models for a banking enterprise. The actual design may vary based on the specific requirements and complexities of the banking system.
