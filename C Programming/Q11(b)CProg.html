Q11(b). Write a C program to convert a positive integer to a roman numeral. Ex. 11 is converted to XI.

Aim: To convert a positive integer to a Roman numeral using a C program. Ex. 11 is converted to XI.

System and Software Requirements: A Computer with any Operating system with GCC compiler installed in it.

Description:
The Roman numeral system has a set of symbols that represent specific values. The symbols and their values are as follows:

  • I = 1
  • V = 5
  • X = 10
  • L = 50
  • C = 100
  • D = 500
  • M = 1000
    • To convert a positive integer to a Roman numeral, we follow the following steps:
    1. Divide the positive integer into thousands, hundreds, tens, and units.
    2. For each of the four parts, find the largest Roman numeral symbol that is less than or equal to the number.
    3. Repeat this process for each of the four parts, making sure to append the symbols in the proper order (units, tens, hundreds, thousands).
    4. Combine the symbols for each part to form the final Roman numeral representation of the positive integer.
    Let's take the example of the positive integer 11 and convert it to its equivalent Roman numeral representation (XI).
    1. Divide 11 into thousands, hundreds, tens, and units. Since 11 is less than 1000, there are no thousands.
    2. For the tens part, find the largest Roman numeral symbol that is less than or equal to 11. In this case, it is X (10).
    3. Repeat the process for the units part. The largest Roman numeral symbol less than or equal to 1 is I.
    4. Combine the symbols X and I to form the final Roman numeral representation of 11, which is XI.
    Expected Results: The program should correctly convert the positive integer 11 to its equivalent Roman numeral representation, XI.
    Conclusion: By following the steps outlined in the theory section, we were able to write a C program to convert a positive integer to its equivalent Roman numeral representation using a simple mathematical example. The program was tested with the positive integer 11 and was able to correctly convert it to its equivalent Roman numeral representation, XI.

    1. Divide 35 into thousands, hundreds, tens, and units. Since 35 is less than 1000, there are no thousands.
    2. For the tens part, find the largest Roman numeral symbol that is less than or equal to 35. In this case, it is XXX (30).
    3. Repeat the process for the units part. The largest Roman numeral symbol less than or equal to 5 is V.
    4. Combine the symbols XXX and V to form the final Roman numeral representation of 35, which is XXXV.

    #include<stdio.h>
    #include<string.h>
    char *intToRoman(int num) {
    static char *ones[] = {"", "I", "II", "III", "IV", "V", "VI", "VII", "VIII", "IX"};
    static char *tens[] = {"", "X", "XX", "XXX", "XL", "L", "LX", "LXX", "LXXX", "XC"};
    static char *hundreds[] = {"", "C", "CC", "CCC", "CD", "D", "DC", "DCC", "DCCC", "CM"};
    static char result[100];
    sprintf(result, "%s%s%s", hundreds[num / 100], tens[(num / 10) % 10], ones[num % 10]);
    return result;
    }
    int main() {
    int num;
    printf("Enter a positive integer: ");
    scanf("%d", &num);
    printf("The Roman numeral representation of %d is: %s\n", num, intToRoman(num));
    return 0;
    }
